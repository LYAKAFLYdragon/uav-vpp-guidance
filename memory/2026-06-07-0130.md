## 关键决策:
1. **env_factory 模式**: BilevelTrainer 不接受单个 env，而是接受 `env_factory` callable。每次评估/收集都创建 fresh env，彻底避免 `current_gains` 状态污染。
2. **Regret 定义**: `regret = 1.0 - max(best_known, current_eval_sr)`。随着 best_known 只增不减，regret 单调非增，符合论文要求。
3. **PPO 接口适配**: `BilevelTrainer` 使用 `select_action` + `store_transition` + `update(next_obs)` 接口，而非用户伪代码中的 `update_from_buffer`/`state_dict()`（PPOAgent 无此方法）。

## 遇到的问题及解决:
1. **`from scripts.run_gain_only_cem import ...` ModuleNotFoundError**: `scripts` 不是 Python package。修复为在 `train_bilevel.py` 中内联定义 `load_config` 和 `build_gain_space`。
2. **PPOAgent 无 `update_from_buffer` / `state_dict`**: 用户伪代码假设了不存在的接口。实际使用 `select_action` + `store_transition` + `update()` 流程。
3. **Regret 单调性**: 初始实现 `regret = 1.0 - eval_sr` 导致 regret 随 eval_sr 波动。修复为 `regret = 1.0 - best_known`，确保单调非增。
