## 关键决策:
1. **Checkpoint 优先级显式化**: 用纯函数实现，避免在 `evaluate_method()` 中隐式拼接路径。source 标签让审计一目了然。
2. **Gains schema 严格但可绕过**: 非 smoke 必须 valid；smoke 允许继续但标记 `gains_schema_valid=false`。避免在 paper-safe 路径下 silent 使用空 gains。
3. **Env current_gains bugfix**: 这是 Stage 8C 遗留问题——metadata 显示 gains 已加载，但 env 未使用。Stage 9A 必须 freeze 端到端语义，所以一并修复并加 integration test 覆盖。
4. **Dummy checkpoint fixture**: 用 `torch.manual_seed(0) + PPOAgent.save()` 在测试里创建确定性权重，避免依赖真实训练产物，保持 CI 可重复。

## 遇到的问题及解决:
1. **Telemetry 未变化**: 初版 integration test 发现 k_los 改变但 telemetry 完全相同。根因是 `CloseRangeTrackingEnv.current_gains` 硬编码为默认 `GuidanceGains()`，忽略 config。修复后 test 通过。
2. **Checkpoint source 断言错误**: smoke manifest 测试中期望 `methods_default`，实际为 `config_method`，因为 stage6f5 配置覆盖了默认 checkpoint。更新测试以反映真实优先级行为。
3. **Unknown method skip 测试空结果导致 matplotlib 崩溃**: 单独传入 bogus 方法会产生空 results，generate_figures 可能失败。改为同时传入 `no_prediction bogus`，让已知方法产生结果。
