## Summary
Finalized the `baseline` branch flyable-baseline fixes, merged into `main`, and tagged `baseline-v1.0`.

## Key Changes Included
- Success-criterion curriculum in `train_no_prediction_vpp_ppo.py` (final hold 1.0s).
- Scenario rebalance: `favorable` own 220 / target 190 m/s.
- Reward tuning: `w_alive`, `w_closing`, `w_overshoot`, plus new `w_boundary` proximity penalty.
- Distance-aware throttle in `LOSRateGuidance` to reduce overshoot.
- Robust `LOSRateGuidance._extract_target_speed()` numpy truth-value fix.
- Deterministic L1 fixed-gain eval path in `train_fixed_gain.py`.
- Smoke gate now validates both L1 and L3.
- CUDA→CPU fallback in `PPOAgent`.

## Known Limitations
- Final strict hold remains 1.0s; 2.0s hold yielded 0% SR at 50k–200k steps and is deferred to future work (`config/success_criteria/strict.yaml` documents it).
- L3 OOB is exactly at the 50% gate threshold. The new `w_boundary` penalty did not reduce OOB within the 50k-step gate, but keeps the gate passing. Further OOB reduction is desirable.
- Pre-existing test failure remains: `tests/test_stage6b_benchmark.py::TestBenchmarkConfig::test_config_has_benchmark_defaults` (unrelated).
